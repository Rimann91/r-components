Overview

All design tokens live in styles/gruvbox_dark_colors.css and are available as CSS custom properties on :root. They automatically swap between light and dark values via @media (prefers-color-scheme: dark) — no class toggling required.

Components never reference global tokens directly in shadow styles. Instead, each component exposes --component-* vars on :host that fall back to the relevant global token (ADR-006). This allows per-instance overrides from outside the shadow DOM.

Typography

Fira Code (monospace) is the only typeface. Font sizes use rem units with a 14px base. Use semantic tokens for size, weight, and line-height rather than raw values.
TokenValueUse for
--font-body "Fira Code", monospace Base font stack — apply to body or component root
--text-xs 0.75rem (10.5px) Labels, captions, nav labels, helper text
--text-sm 0.875rem (12.25px) Secondary text, table cells, input values
--text-base 1rem (14px) Default body copy, paragraph text
--text-lg 1.125rem (15.75px) Slightly prominent body text or subheadings
--text-xl 1.25rem (17.5px) Section headings (h4/h5)
--text-2xl 1.5rem (21px) Page-level headings (h3)
--text-3xl 1.75rem (24.5px) Major headings (h2)
--font-weight-normal 400 Default body weight
--font-weight-medium 500 Slightly emphasised labels
--font-weight-semibold 600 Subheadings, button labels
--font-weight-bold 700 Headings, strong emphasis
--line-height-tight 1.25 Headings and single-line display text
--line-height-base 1.5 Body copy and multi-line text
Border Radius

A single global token. Components expose --component-radius on :host that falls back to this value, allowing per-instance overrides. Some components intentionally deviate (e.g. r-badge is always pill-shaped at 9999px).
TokenValueUse for
--radius4pxDefault corner radius for all components (buttons, cards, inputs, etc.)
Transitions

Full shorthand values including easing. Use only with the transition: CSS property. Never embed in animation: shorthand — the easing value will conflict and the browser will silently drop the entire rule.
⚠ Animation pitfall: --transition-base expands to 250ms ease. Writing animation: spin var(--transition-base) linear infinite produces two easing values and the browser drops the rule. Use a bare duration token instead: --my-duration: 250ms then animation: spin var(--my-duration) linear infinite.
TokenValueUse for
--transition-fast 150ms ease Hover states, focus rings, quick micro-interactions
--transition-base 250ms ease Most UI transitions — expanding panels, colour shifts
--transition-slow 400ms ease Larger motions — modals, drawers, page transitions
Shadows

Use to convey elevation. Reserve larger shadows for elements that float above content (dropdowns, modals, tooltips). Avoid on flat/inline elements.
TokenValueUse forPreview
--shadow-sm 0 1px 2px rgba(0,0,0,.12) Subtle lift — cards, inputs, badges
--shadow-base 0 2px 4px rgba(0,0,0,.16) Default elevation — dropdowns, popovers
--shadow-lg 0 4px 8px rgba(0,0,0,.20) High elevation — modals, overlays, toasts
Focus

Used to indicate keyboard focus. Always use --focus-ring as the box-shadow value on interactive elements — never roll a custom focus style.
TokenValueUse for
--color-focus var(--color-primary-blue) Focus indicator colour — resolves to the Gruvbox blue
--focus-ring 0 0 0 2px var(--color-focus) Apply as box-shadow on :focus-visible
Z-index Scale

Layering scale for floating elements. Use the named token that matches the role of the element — don't use arbitrary numbers.
TokenValueUse for
--z-dropdown 100 Dropdowns, select menus, autocomplete lists
--z-modal 200 Modal dialogs and their backdrops
--z-tooltip 300 Tooltips — always above modals
